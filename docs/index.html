The problem

AI tools are a data exfiltration risk

Every time you accept a suggestion or send a chat message, your editor context travels to a third-party server. Most developers have no idea what's in that payload.

AI tools see everything

Your editor context includes open files, unsaved buffers, git history, and environment variables — all of it gets serialised into the prompt and sent to third-party LLM servers.

Secrets leak silently

GitGuardian 2026: 29 million secrets detected on public GitHub alone. AI-assisted commits leak credentials at 2× the rate of human developers — and you never get an error when it happens.

2× higher leak rate with AI

No visibility

Without a proxy layer, you have zero insight into what leaves your machine, which rules would have triggered, or how much token budget is being consumed across your team.

How it works

Silent interception, zero friction

One environment variable routes all AI traffic through Prompt Guard. Your tools behave exactly the same — minus the data you didn't mean to share.

Connect

Route

Set HTTPS_PROXY=localhost:8080 — your AI client now routes through Prompt Guard automatically.

Inspect

Inspect

Prompt Guard decrypts each request and runs 230 detection rules across the full prompt payload — gitleaks patterns plus hand-rolled PII rules.

Act

Block or Redact

Matched secrets are either blocked (request rejected) or redacted ([REDACTED] substituted) before forwarding.

Audit

Audit

Every request is logged to SQLite with matched rules, token usage, latency, and model. Fully visible in the live dashboard.

Features

Everything you need, nothing you don't

Built to stay out of the way until something important is about to leave your machine.

230 Detection Rules

Gitleaks patterns for API keys, tokens, and credentials — plus hand-rolled rules for SSNs, credit card numbers, database connection strings, and bearer tokens. Configurable per-rule.

Block & Redact Modes

Block stops the request entirely — the AI never sees it. Redact scrubs the matched value and forwards the sanitised prompt so your workflow continues uninterrupted.

Live Dashboard

Real-time feed of intercepted prompts, matched rules, latency stats, token usage, and model tracking — all at localhost:7778. No external service required.

Agent Mode

Switch all rules to redact so long-running autonomous agents are never hard-blocked mid-task. Secrets are still scrubbed, but the agent loop continues without interruption.

Works Everywhere

Claude Code, GitHub Copilot (Pro / Business / Enterprise), VS Code, curl — any HTTP_PROXY-aware tool works without modification.

Single Binary, No Cloud

Go binary with zero runtime dependencies. SQLite audit log stays on disk. Runs entirely on localhost — your prompts never touch a Prompt Guard server because there isn't one.

Quick start

Up and running in three steps

No Docker. No daemon. No config file required to start.

1

Build & run

Clone the repo and build the Go binary. Prompt Guard starts immediately and listens on port 8080.

git clone https://github.com/chaudharydeepak/prompt-guard cd prompt-guard go build -o prompt-guard . ./prompt-guard # Proxy listening on :8080 # Dashboard ready at http://localhost:7778
2

Set proxy environment variables

In a new terminal (or add to your shell profile), export the proxy variables. For Claude Code, also set NODE_EXTRA_CA_CERTS so the TLS certificate is trusted.

# Shell (bash / zsh) export HTTP_PROXY=http://localhost:8080 export HTTPS_PROXY=http://localhost:8080 # Claude Code — trust the generated cert export NODE_EXTRA_CA_CERTS=~/.prompt-guard/ca.crt
3

Open the dashboard

Open your browser and head to the dashboard. Every intercepted prompt appears in real time. Adjust rules, switch to Agent Mode, or export the audit log.

# macOS open http://localhost:7778 # Linux xdg-open http://localhost:7778
